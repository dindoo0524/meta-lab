# Claude Code Workflow — 성장톡

## 개요

이 프로젝트는 Claude Code를 주요 개발 파트너로 사용한다.
아래는 효과적인 협업을 위한 가이드라인.

## 작업 방식

### Plan → Execute → Document
1. 작업 전에 계획 수립 (Plan Mode 활용)
2. 계획에 따라 구현
3. 구현 후 문서 업데이트

### 커밋 규칙
- 각 의미 있는 변경 후 커밋
- 접두사: `feat:`, `fix:`, `docs:`, `style:`, `chore:`
- 커밋 메시지는 "왜"에 집중

### 스코프 관리
- `docs/mvp-scope.md`를 스코프 계약서로 사용
- 새 기능 요청은 스코프 문서와 대조
- 범위 밖이면 `docs/future-ideas.md`에 기록

## CLAUDE.md 활용

- 프로젝트 컨텍스트와 제약 조건 정의
- 어시스턴트 캐릭터 설정 (메타몽)
- 기술 스택 및 참조 코드베이스 명시

## 주의사항

- 오버엔지니어링 금지
- 작은 스코프 우선
- 불확실하면 구현하지 않음
- 사용자가 자주 확인할 수 없으므로 자율적이되 보수적으로 작업
- 문서화를 통해 나중에 리뷰할 수 있게 유지
