# META Lab — 포트폴리오

> AI를 풀어주는 대신, 제약했더니 더 좋은 학습이 일어났다.

---

## 시작

범용 AI에게 "오늘 하루 어땠어?"라고 물으면, 길고 따뜻한 동기부여 문단이 돌아옵니다. 스페인어를 가르쳐달라고 하면, 문법 설명을 쏟아냅니다. 내 생각에 도전해달라고 하면, 정중하게 동의합니다.

저는 반대로 해봤습니다.

AI에게 제약을 걸었습니다. 주제를 제한하고, 응답 방식을 통제하고, 할 수 있는 것을 줄였습니다. 그랬더니 오히려 — 더 깊은 대화, 더 집중된 학습, 더 날카로운 사고가 일어났습니다.

META Lab은 그 실험의 기록입니다.

---

## 핵심 아이디어

**제약이 집중을 만든다.**

이건 직관에 반하는 주장입니다. AI의 강점은 무한한 가능성인데, 왜 제한하냐고요.

하지만 학습에는 경계가 필요합니다. 성찰에는 구조가 필요합니다. 사고에는 저항이 필요합니다.

META Lab의 각 실험은 하나의 가설입니다:
- "AI가 **설명을 못 하게** 하면, 사용자가 스스로 더 생각하지 않을까?"
- "AI가 **한 가지만 물어보게** 하면, 대화가 더 깊어지지 않을까?"
- "AI가 **절대 편을 안 들게** 하면, 사용자가 더 균형 잡힌 판단을 하지 않을까?"

답은 — 그렇습니다. 놀라울 정도로.

---

## 개발 방식

이 프로젝트는 **AI 주도 개발**로 만들었습니다.

"AI가 코드 좀 짜줬다"는 수준이 아닙니다. Claude Code를 실시간 구현 파트너로 두고, 아키텍처 결정부터 프롬프트 엔지니어링, 피벗 판단, 문서화까지 함께 진행했습니다.

실제로 이렇게 일했습니다:
- 방향을 대화로 정하고, 계획을 세우고, 바로 구현
- 3-step 고정 플로우 → 자유 채팅으로 피벗: 대화 중에 결정
- 단일 서비스 → 실험 저장소로 재정의: 대화 중에 결정
- 6개 실험, AI 스트리밍, 히스토리, 스트릭 시스템, Supabase 연동 — 몇 번의 집중 세션으로 완성

AI가 개발 실력을 대체하지는 않습니다. 하지만 **뭘 만들어야 하는지 아는 사람**에게는 실행력을 극적으로 증폭시킵니다.

---

## 시스템 설계

### 프롬프트 레이어링

AI의 모든 응답은 3개 레이어의 합성입니다:

```
┌─────────────────────────┐
│  실험 프롬프트           │  무엇에 대해, 어떤 규칙으로, 어떤 톤으로
├─────────────────────────┤
│  스타일 프롬프트         │  Iris(차분) / Milo(따뜻) / Noah(직설)
├─────────────────────────┤
│  제약 프롬프트           │  Free(자유) / Guided(유도) / Strict(엄격)
└─────────────────────────┘
```

하나의 실험을 **9가지 조합**으로 경험할 수 있습니다.

Noah + Strict 모드의 스페인어 수업은 칼같은 드릴링.
Milo + Free 모드의 같은 수업은 따뜻한 격려 대화.
같은 콘텐츠, 완전히 다른 경험.

### ExperimentConfig

모든 실험은 하나의 설정 객체로 정의됩니다. 새 실험 추가 = config 파일 1개 + 페이지 2개. 허브는 레지스트리에서 자동 렌더링됩니다.

### 저장소

이중 저장 구조:
- **Supabase** — 기기 간 동기화 (설정 시)
- **localStorage** — 즉시 반응, 안정적 폴백 (항상)
- **device_id** — 인증 없이 UUID로 사용자 식별

## 주요 실험들

### 성장톡
하루를 돌아보는 대화. AI는 간결하고 관찰적인 질문만 합니다. 설교 안 합니다. 동기부여 문단 안 씁니다. 공감 스팸 안 합니다. 그냥 — 좋은 질문 하나.

### 스페인어 튜터
칸아카데미의 칸미고(Khanmigo)에서 영감 받았습니다. 답을 안 줍니다. 퀴즈, 힌트, 격려만. 특별한 맥락이 있습니다: 10월 스페인 신혼여행 준비.

### 하루 한 줄
가장 극단적인 실험. 오늘을 한 문장으로 적으면, AI가 한 가지만 물어봅니다. 끝. 설명 없음. 조언 없음. 요약 없음. 짧음에서 오는 깊이가 있습니다.

### 역질문
5 Whys 기법. 당신이 뭘 말하든, AI는 "왜?"만 묻습니다. 담담하게, 집요하게, 끝까지. 설명 안 합니다. 동의 안 합니다. 그냥 계속 파고듭니다.

### 감정 번역기
"짜증났어"라고 하면 AI는 "힘들었겠다"라고 하지 않습니다. 대신 물어봅니다: *무력감에 가까운가요, 서운함에 가까운가요, 지침에 가까운가요?* 감정에 정확한 이름을 붙이는 것 — 그것만이 목적입니다.

### 양면 사고
고민을 던지면 AI가 찬성과 반대를 분석합니다. 절대 편을 안 듭니다. 항상 끝에 묻습니다: "당신은 어떻게 생각하세요?" 제약: 완전한 중립.

## 배운 것들

**제약은 제한이 아니라 기능이다.**
가장 많이 제약된 실험이 가장 좋은 경험을 만들었습니다. "하루 한 줄"(1문장 입력, 1질문 반환)이 어떤 자유 대화보다 강력합니다.

**캐릭터는 장식이 아니라 UX다.**
같은 실험이 Iris와 Noah에서 완전히 다르게 느껴집니다. 성격은 기능입니다.

**AI 주도 개발은 — 비전이 있을 때 작동한다.**
Claude Code가 제 판단을 대신하지 않았습니다. 실행력을 증폭시켰습니다. 속도 향상은 실제입니다. 하지만 뭘 만들지 아는 사람에게만.

**기획보다 프로토타이핑.**
이 프로젝트의 모든 중요한 방향 전환은 기획 문서가 아니라, 프로토타입을 *써보면서* 일어났습니다. 먼저 만들고, 나중에 다듬기.
